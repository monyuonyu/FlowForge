# FlowForge

**検証可能なイベント駆動 離散事象シミュレータ（Discrete-Event Simulation, DES）— Godot 4.x / GDScript**

FlexSim のような「モノの流れ」を 3D で組み立て・可視化・実験できる DES です。ただ動くだけでなく、
**出てくる数字を待ち行列理論や保存則で裏づける**ことを設計の中心に据えています。Source → Queue →
Processor → Conveyor → Sink とアイテムが流れ、作業者や搬送機（共有資源）を奪い合い、故障・段取り・
シフト・AGV の交通渋滞まで含めて挙動が決まります。

> レビュー用モデル（fable5）による第9回評価: **9.9/10**（推移 3 → 6 → 7 → 9 → 9.5 → 9.7 → 9.8 → 9.9）。
> 「検証可能性で勝り、規模と統合度で劣る、独立した小型 DES」。完全な FlexSim 同等（10/10）は
> エコシステム（教材・コミュニティ・20年分のモデル資産・商用サポート）を含むためコードだけでは
> 原理的に到達不可、という前提のもとでの評価です。

## 何が「本物」か

- **イベント駆動エンジン** — 二分ヒープのイベントカレンダー。時間ステップ式ではなく、`run_until` で
  瞬時に長時間を回せる。
- **完全な再現性** — シード管理された設備別独立乱数ストリーム（FNV-1a 安定ハッシュ）。**同一シード＝
  バイト単位で同一結果**。
- **外部真値との突合** — M/M/1・M/M/c の稼働率 / Lq / Wq が待ち行列理論と信頼区間内で一致。
  リトルの法則も数値的に整合。＝出てくる数字を信頼できる。
- **保存則** — created = Σsink + WIP を全機能で維持（二重計上・アイテム消失なし）。
- **自己テスト網** — ヘッドレスで **81 種の検証マーカー**（`[determinism] identical=true` /
  `[conserve] ok=true` / `[mm1]`/`[mmc]` 理論一致 / `[pf-travel-congest]` 等）。新機能を足すたびに
  対応マーカーを追加し、決定論と保存則が緑のまま維持されることを担保。

## 主な機能

- **フロー部品**: Source / Queue / Processor / Conveyor / Sink / Combiner / Separator / Rack（立体倉庫）
- **資源**: 作業者プール（FIFO/nearest ディスパッチ・シフト）、搬送機プール（複数積載・積降時間・優先度）
- **AGV 経路網**: Dijkstra 最短路 ＋ **交通干渉**（エッジ占有セマフォ・FIFO 解放待ち・単線双方向の
  デッドロック回避）＝渋滞を再現
- **プロセスフロー（Process Flow）**: トークンベースのロジック層。source/delay/assign/decide/batch/
  acquire・release/**travel・load・unload** 等をイベントカレンダー上に。実 3D モデル・実資源を編成し、
  **AGV 交通干渉と合成した所要時間が返る**（FlexSim 中核体験）。JSON 保存/読込・spec 検査（lint）対応。
- **統計・分析**: 時間加重 WIP、Lq/Wq、滞留ヒストグラム、状態ガント、Welch 法 warmup 推定、
  目標精度ベースの反復数決定、HTML/CSV レポート
- **実験**: N レプリケーション＋Student-t 95%CI、シナリオ実験（CRN 共通乱数・対比較 CI）、
  パラメータ掃引、**最適化（OptQuest 相当: grid/random/hill）**、フレーム分割の**バックグラウンド実行**（中断・進捗）
- **分布**: const/exp/uniform/normal(切断)/triangular/lognormal/weibull/gamma/empirical
- **CAD 系エディタ**: 縮尺グリッド・スナップ・メジャー・正射投影/ビュープリセット・数値トランスフォーム・
  直交拘束
- **モデル構築 UX**: クリック配置パレット・右クリックメニュー・複製(Ctrl+D)・キーボードショートカット・
  Undo/Redo・ドラッグ配線
- **組み込みスクリプト**: GDScript 実行時コンパイル（FlexScript 相当）、読込時セキュリティ確認

## 使い方

```bash
# Godot 4.x（検証は 4.7.1）で project.godot をインポートし F5 で実行
# 左上「▶開始」／「🔧編集モード」で構築

# ヘッドレス自己テスト（全マーカーを標準出力へ）
SIM_HEADLESS_TEST=1 godot --headless --path .

# スクリーンショット（要 xvfb か実表示）
SIM_SHOT=1 godot --path .
```

自己テストは `scripts/Main.gd` の `_run_headless_test()`。新機能を足したら対応する `[xxx]` マーカーを
追加し、`[determinism]` と `[conserve]` が緑のままかを必ず確認する運用です。
